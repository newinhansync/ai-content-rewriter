# Part 1: 프로젝트 개요 및 요구사항

## 1.1 프로젝트 명
**AI Content Rewriter for WordPress** (가칭: WP-AI-Rewriter)

## 1.2 프로젝트 목적
외부 URL 콘텐츠 또는 직접 입력한 텍스트를 AI를 활용하여 블로그 게시글로 자동 변환하는 워드프레스 플러그인 개발

## 1.3 핵심 기능 요구사항

### 1.3.1 콘텐츠 입력 방식
| 입력 유형 | 설명 |
|----------|------|
| URL 입력 | 특정 웹페이지 URL을 입력하면 콘텐츠 자동 추출 |
| 텍스트 붙여넣기 | 직접 텍스트를 복사/붙여넣기로 입력 |
| 다중 URL 지원 | 여러 URL을 배치로 처리 |

### 1.3.2 AI 처리 기능
| 기능 | 설명 |
|-----|------|
| 콘텐츠 재작성 | 원본 콘텐츠를 새로운 블로그 글로 재구성 |
| 번역 및 로컬라이제이션 | 영어 → 한글 등 다국어 변환 |
| 커스텀 프롬프팅 | 사용자 정의 지시사항으로 글 스타일 조정 |
| 긴 콘텐츠 처리 | 장문 콘텐츠도 분할 처리하여 완전한 결과물 생성 |

### 1.3.3 지원 AI 모델
- **ChatGPT-5** (OpenAI)
- **Gemini-3** (Google)
- 향후 확장 가능한 모듈러 구조

### 1.3.4 자동화 기능
| 기능 | 설명 |
|-----|------|
| 스케줄링 | 예약된 시간에 자동으로 콘텐츠 생성 |
| 배치 처리 | 여러 URL을 순차적으로 자동 처리 |
| 크론 작업 통합 | WordPress Cron 활용 |

### 1.3.5 게시글 메타데이터 자동 생성
- 제목 (Title)
- 슬러그 (Slug)
- 카테고리 (Categories)
- 태그/키워드 (Tags/Keywords)
- 메타 설명 (Meta Description)
- 특성 이미지 제안 (Featured Image Suggestion)
- SEO 메타 태그

## 1.4 사용자 시나리오

### 시나리오 1: 단일 URL 콘텐츠 변환
```
1. 관리자가 플러그인 페이지 접속
2. 영문 기사 URL 입력
3. 출력 언어: 한국어 선택
4. AI 모델: ChatGPT-5 선택
5. 커스텀 프롬프트 입력 (선택)
6. "생성" 버튼 클릭
7. AI가 콘텐츠 추출 → 번역 → 재구성
8. 미리보기 확인
9. 게시글로 발행 또는 초안 저장
```

### 시나리오 2: 스케줄 기반 자동화
```
1. RSS 피드 또는 URL 목록 등록
2. 스케줄 설정 (예: 매일 오전 9시)
3. 처리 옵션 설정 (언어, AI 모델, 프롬프트)
4. 자동 발행 또는 초안 저장 선택
5. 시스템이 예약된 시간에 자동 처리
6. 관리자에게 알림 발송
```

## 1.5 비기능적 요구사항

### 1.5.1 성능
- 단일 URL 처리: 60초 이내 완료
- 긴 콘텐츠 (10,000자 이상): 청크 분할 처리
- API 호출 최적화로 비용 효율성 확보

### 1.5.2 보안
- API 키 암호화 저장
- 입력값 새니타이징
- CSRF 보호
- 권한 기반 접근 제어 (관리자/편집자)

### 1.5.3 호환성
- WordPress 6.0 이상
- PHP 8.0 이상
- MySQL 5.7 이상
- 주요 테마/플러그인 호환성

### 1.5.4 확장성
- 새로운 AI 모델 추가 용이
- 프롬프트 템플릿 시스템
- 훅(Hook) 및 필터(Filter) 제공

## 1.6 제약사항 및 고려사항

### 1.6.1 저작권 관련
- 원본 콘텐츠 저작권 준수 안내
- 콘텐츠 변형 정도 옵션 제공
- 출처 표기 옵션

### 1.6.2 API 비용
- AI API 호출 비용 발생
- 토큰 사용량 모니터링 기능
- 사용량 제한 설정 가능

### 1.6.3 품질 관리
- AI 생성 콘텐츠 검토 워크플로우
- 초안 상태로 저장 옵션
- 수동 검토 후 발행 권장

## 1.7 성공 기준

|-----|------|
